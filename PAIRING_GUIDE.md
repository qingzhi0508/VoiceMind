# VoiceMind 配对流程说明

## 配对步骤

### 1. macOS 端操作

1. **启动服务**：
   - 打开 VoiceMind 应用
   - 在「状态」标签页点击「启动服务」按钮
   - 等待服务启动成功（控制台会显示端口号）

2. **开始配对**：
   - 点击「开始配对」按钮
   - 会弹出配对窗口，显示 **6位数字配对码**
   - 例如：`123456`
   - 配对码有效期 2 分钟

### 2. iOS 端操作

1. **发现设备**：
   - 打开 VoiceMind iOS 应用
   - 应用会自动搜索局域网内的 Mac
   - 在设备列表中找到你的 Mac（显示 Mac 的名称）

2. **输入配对码**：
   - 点击你的 Mac 设备
   - 在弹出的输入框中输入 macOS 端显示的 6 位数字
   - 点击「确认」

3. **配对成功**：
   - iOS 端显示「已配对」
   - macOS 端配对窗口自动关闭
   - 状态页显示已配对设备名称

## 配对流程（技术细节）

```
macOS                           iOS
  |                              |
  | 1. 启动服务                   |
  |    - WebSocket Server        |
  |    - Bonjour 广播            |
  |                              |
  |                              | 2. 发现服务
  |                              |    - Bonjour 浏览
  |                              |    - 显示设备列表
  |                              |
  | 3. 开始配对                   |
  |    - 生成 6 位码              |
  |    - 显示配对窗口             |
  |                              |
  |                              | 4. 用户输入配对码
  |                              |
  |    <--- pairConfirm ------   | 5. 发送配对确认
  |    (shortCode, iosName, iosId)|
  |                              |
  | 6. 验证配对码                 |
  |    - 生成 sharedSecret       |
  |    - 保存到 Keychain         |
  |                              |
  |    ---- pairSuccess ----->   | 7. 发送配对成功
  |    (sharedSecret)            |
  |                              |
  |                              | 8. 保存配对信息
  |                              |    - 保存到 Keychain
  |                              |
  | 9. 配对完成                   | 10. 配对完成
  |    - 关闭配对窗口             |     - 显示已配对
```

## 常见问题

### Q1: iOS 端找不到 Mac 设备

**可能原因**：
1. macOS 端服务未启动
2. 两个设备不在同一局域网
3. 防火墙阻止了 Bonjour 广播

**解决方案**：
1. 确认 macOS 端「状态」页显示「服务运行中」
2. 确认两个设备连接到同一 WiFi
3. 在 macOS 系统设置中检查防火墙设置

### Q2: 输入配对码后没有反应

**可能原因**：
1. 配对码输入错误
2. 配对码已过期（2分钟）
3. 网络连接问题

**解决方案**：
1. 仔细核对配对码（6位数字）
2. 如果超过2分钟，重新开始配对
3. 检查网络连接

### Q3: 配对成功但无法使用

**可能原因**：
1. macOS 端缺少辅助功能权限
2. 快捷键监听未启动

**解决方案**：
1. 在「权限」标签页检查权限状态
2. 确认「状态」页显示「快捷键监听：已启用」

## 调试配对问题

### macOS 端调试

1. 打开「调试」标签页
2. 查看应用信息和网络状态
3. 查看 Xcode 控制台日志：
   ```
   ✅ WebSocket server started on port XXXX
   ✅ Bonjour service: _voicerelay._tcp
   🔐 开始配对，代码: XXXXXX
   ```

## 配对后的使用

1. **触发语音识别**：
   - 在 Mac 上按下快捷键（默认：Option + Space）
   - iOS 端开始录音
   - 松开快捷键停止录音

2. **文本注入**：
   - iOS 端识别完成后自动发送结果
   - Mac 端将文本注入到当前应用

3. **解除配对**：
   - macOS 端：在「状态」页点击「解除配对」
   - iOS 端：在设置中点击「解除配对」

## 安全说明

- 配对码只在局域网内传输
- 配对成功后使用 HMAC-SHA256 验证所有消息
- 共享密钥（sharedSecret）安全存储在 Keychain 中
- 每次配对生成新的随机密钥

## 网络要求

- 两个设备必须在同一局域网
- macOS 端需要开放 8000-9000 范围内的一个端口
- 支持 Bonjour/mDNS 服务发现
- 使用 TCP 连接（非 WebSocket）
