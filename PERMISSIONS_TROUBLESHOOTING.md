# 辅助功能权限配置指南

## 问题：辅助功能列表中找不到应用

这是 macOS 开发环境的常见问题。每次在 Xcode 中构建应用，macOS 可能认为这是一个"新"应用。

## 解决方案

### 方法 1：使用调试视图（推荐）

1. 在应用中切换到「调试」标签页
2. 点击「2️⃣ 请求辅助功能权限」按钮
3. 这会触发系统弹窗，应用会自动添加到权限列表
4. 如果系统设置打开了，按照提示操作

### 方法 2：手动添加应用

1. 打开「系统设置」→「隐私与安全性」→「辅助功能」
2. 点击左下角的「🔒」图标解锁（需要输入密码）
3. 点击「+」按钮
4. 导航到应用位置：
   ```
   ~/Library/Developer/Xcode/DerivedData/VoiceRelay-xxx/Build/Products/Debug/VoiceRelayMac.app
   ```
   提示：在调试标签页可以看到完整的应用路径
5. 选择 VoiceRelayMac.app 并添加
6. 勾选启用

### 方法 3：使用终端命令

```bash
# 查看当前权限状态
tccutil reset Accessibility com.yourcompany.VoiceRelayMac

# 注意：Bundle ID 可能不同，在调试标签页查看实际的 Bundle ID
```

## 为什么会出现这个问题？

1. **开发环境特性**：Xcode 每次构建会生成新的应用路径
2. **应用签名**：开发版本的签名与发布版本不同
3. **macOS 安全机制**：macOS 通过应用路径和签名识别应用

## 验证权限是否生效

在调试标签页：
1. 点击「1️⃣ 检查权限」
2. 查看权限状态：
   - ✅ 已授予 - 权限配置成功
   - ❌ 未授予 - 需要继续配置

## 发布版本的处理

发布应用时，需要：
1. 使用 Developer ID 签名
2. 进行公证（Notarization）
3. 首次运行时引导用户授权
4. 发布后的应用路径固定，不会有这个问题

## 常见问题

### Q: 点击「请求权限」后没有反应？
A: 检查控制台日志，可能是应用签名问题。尝试在 Xcode 中配置签名。

### Q: 授予权限后应用仍显示「未授予」？
A: 点击「检查权限」刷新状态，或重启应用。

### Q: 每次重新构建都需要重新授权？
A: 这是开发环境的正常现象。可以：
   - 使用固定的 Bundle Identifier
   - 使用相同的签名证书
   - 或者每次重新授权（开发阶段）

## 输入监控权限

输入监控权限的配置方式相同：
1. 在调试标签页点击「3️⃣ 请求输入监控权限」
2. 或在系统设置中手动添加到「输入监控」列表

## 需要帮助？

如果以上方法都不行：
1. 查看调试标签页的「应用信息」部分
2. 记录 Bundle ID 和应用路径
3. 检查 Xcode 的签名配置
4. 查看控制台日志中的错误信息
