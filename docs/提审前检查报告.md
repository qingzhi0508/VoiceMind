**建议**：正式发布前改为反域名格式，如 `cayden.VoiceMind`

修复方式（Xcode 中）：

2. 改为 `cayden.VoiceMind`（两个平台统一）
3. 在 Apple Developer 后台确认 App ID 已注册

### 问题 2：iOS 无专属 Entitlements 文件

**现状**：iOS TARGET 没有 `.entitlements` 文件（macOS 有）
**建议**：确认 iOS 端是否需要特殊权限（如 Push Notification、Keychain Sharing）

若后续需要，添加 `VoiceMindiOS.entitlements`：
```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>keychain-access-groups</key>
    <array>
        <string>$(AppIdentifierPrefix)$(PRODUCT_BUNDLE_IDENTIFIER)</string>
    </array>
</dict>
</plist>
```

## 五、建议补充的审核备注

在 App Store Connect 的「审核备注」中可填写：

```
VoiceMind 是一款语音输入效率工具，通过局域网将 iPhone 的语音识别能力投射到 Mac。

主要功能：
- 使用 iPhone 作为麦克风，Mac 端实时显示语音识别结果
- 6 位数字配对码 + HMAC-SHA256 加密保护隐私
- 支持中文（普通话）和英文语音识别
- 识别结果本地保存，不上传任何第三方服务器

隐私说明：
- 本应用不收集、不上传任何用户数据
- 语音识别由 Apple Speech 框架在本地处理
- 配对信息存储在系统 Keychain
- 通信使用 HMAC-SHA256 加密

权限说明：
- 麦克风权限：用于采集语音
- 语音识别权限：用于将语音转为文字
- 本地网络权限：用于 iPhone 与 Mac 之间建立局域网连接
```

## 六、构建验证命令

```bash
# iOS Release 构建
cd /Users/cayden/Data/my-data/voicemind
xcodebuild -workspace VoiceMind.xcworkspace \
  -scheme VoiceMindiOS \
  -configuration Release \
  -destination 'generic/platform=iOS' \
  -archivePath build/VoiceMindiOS.xcarchive \
  archive

# macOS Release 构建
xcodebuild -workspace VoiceMindMac.xcworkspace \
  -scheme VoiceMindMac \
  -configuration Release \
  -archivePath build/VoiceMindMac.xcarchive \
  archive

# 导出 iOS .ipa
xcodebuild -exportArchive \
  -archivePath build/VoiceMindiOS.xcarchive \
  -exportOptionsPlist exportOptions_iOS.plist \
  -exportPath build/

# 导出 macOS .pkg
xcodebuild -exportArchive \
  -archivePath build/VoiceMindMac.xcarchive \
  -exportOptionsPlist exportOptions_macOS.plist \
  -exportPath build/
```
