# 二维码扫描功能测试指南

## 已实现的功能

### iOS 端新增文件
1. ✅ `QRCodeScannerController.swift` - 二维码扫描控制器

2. ✅ `CameraPreview.swift` - 相机预览视图

3. ✅ `QRCodeScannerView.swift` - 完整的扫描界面
   - 相机预览

4. ✅ `Info.plist` - 添加相机权限描述

## 在 Xcode 中添加文件

### 1. 添加 iOS 扫描器文件

在 VoiceRelayiOS 项目中：

1. 创建 `Scanner` 文件夹：
   - 右键点击 `VoiceRelayiOS` → New Group
   - 命名为 `Scanner`

2. 添加文件到 Scanner 文件夹：
   - `QRCodeScannerController.swift`
   - `CameraPreview.swift`

3. 确认 `QRCodeScannerView.swift` 在 `Views` 文件夹中

4. 确保所有文件都勾选了 VoiceRelayiOS target

### 2. 验证 Info.plist

确认 `NSCameraUsageDescription` 已添加：
```xml
<key>NSCameraUsageDescription</key>
<string>VoiceMind 需要使用相机扫描二维码以连接到您的 Mac</string>
```

## 完整测试流程

### 准备工作

1. **macOS 端**：
   - 构建并运行 VoiceRelayMac
   - 在「状态」标签页点击「启动服务」
   - 点击「开始配对」
   - 记录显示的信息：
     - 二维码（用于扫描）
     - IP 地址：`_______________`
     - 端口：`_______________`
     - 配对码：`_______________`

2. **iOS 端**：
   - 构建并运行 VoiceRelayiOS
   - 确保 iOS 设备和 Mac 在同一 WiFi 网络

### 测试方式 1：二维码扫描（推荐）

1. **iOS 端操作**：
   - 点击「扫码连接 Mac」
   - 首次使用会请求相机权限，点击「允许」
   - 将 iPhone 对准 Mac 屏幕上的二维码
   - 等待自动识别（会震动提示）

2. **预期结果**：
   - ✅ 相机预览正常显示
   - ✅ 扫描到二维码后自动连接
   - ✅ 连接成功后弹出配对码输入界面
   - ✅ 显示已连接的 Mac 信息

3. **输入配对码**：
   - 输入 Mac 显示的 6 位数字
   - 点击「配对」

4. **配对成功**：
   - ✅ iOS 显示「已配对」
   - ✅ macOS 配对窗口自动关闭
   - ✅ 两端都显示配对成功

### 测试方式 2：手动输入（备用）

1. **iOS 端操作**：
   - 点击「手动输入连接」
   - 输入 IP 地址和端口
   - 点击「连接」
   - 连接成功后输入配对码
   - 点击「配对」

2. **预期结果**：
   - ✅ 连接成功后自动切换到配对码输入
   - ✅ 配对成功后返回主界面

## 常见问题排查

### Q1: 相机无法启动

**可能原因**：
- 没有相机权限
- 模拟器不支持相机

**解决方案**：
1. 检查 Info.plist 中是否有 `NSCameraUsageDescription`
2. 在设置中检查 VoiceMind 的相机权限
3. 使用真机测试（模拟器相机功能有限）

### Q2: 扫描不到二维码

**可能原因**：
- 二维码太小或模糊
- 光线不足
- 相机对焦问题

**解决方案**：
1. 确保 Mac 屏幕亮度足够
2. 将 iPhone 距离屏幕 15-30cm
3. 保持稳定，等待自动对焦
4. 使用手动输入作为备用方案

### Q3: 扫描后无法连接

**可能原因**：
- 两个设备不在同一网络
- macOS 防火墙阻止
- 端口被占用

**解决方案**：
1. 确认两个设备连接到同一 WiFi
2. 检查 macOS 防火墙设置
3. 查看控制台日志确认具体错误

### Q4: 配对码验证失败

**可能原因**：
- 配对码输入错误
- 配对码已过期（2分钟）

**解决方案**：
1. 仔细核对 6 位数字
2. 如果超时，重新开始配对
3. 查看 macOS 控制台确认收到的配对码
